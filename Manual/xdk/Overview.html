2 Overview

The XDG Development Kit (XDK) is a grammar development environment for the grammar formalism Extensible Dependency Grammar (XDG).

2.1 XDG

An XDG grammar is a description of a set of dependency multigraphs. A dependency multigraph is a set of dependency graphs over the same string of words. The words correspond one-to-one to the nodes in the multigraph. The individual dependency graphs are called dimensions, and the nodes of the dimensions are connected by labeled edges. Dependency multigraphs are a modular way of representing multiple aspects of a linguistic analysis. For example, a multigraph can consist of two dimensions: one representing the grammatical functions, and another representing the topological structure. Here is an analysis of the German subordinate clause (dass) Maria versucht einen Mann zu lieben along these lines: the dimension of grammatical functions is shown on top and the dimension of topological structure below. The modularity of XDG paves the way for a divide-and-conquer strategy to grammatical description. The problem of modeling all linguistic aspects of a grammar and their interactions is broken down into multiple smaller problems:

  1. modeling the individual linguistic aspects independently of each other, each on a separate dimension and
  2. modeling the interactions between them.

This strategy stands in stark contrast to grammar formalisms where the linguistic aspects are modeled in a more interlocked manner, and all their interactions have to be taken care of at the same time. The XDG strategy can lead to significantly simplified and less error-prone grammar development. In addition, it allows for simple yet powerful linguistic models.

An XDG grammar consists of the following ingredients:

  1. a multigraph type determining the basic properties of the multigraphs licensed by the grammar: its dimensions, words, edge labels and attributes.
  2. a set of principles constituting the well-formedness conditions of the grammar.
  3. a lexicon determining specific properties of nodes associated with specific words

2.2 XDK

The XDK is a grammar development environment for XDG. In the XDK, grammars are written as XDK grammar files. Grammar files include definitions of the multigraph type and the lexicon, and references to the principles. Principles are implemented using finite-set constraint programming in Mozart/Oz. Starting with version 1.6.33, the XDK includes PrincipleWriter (PW), developed by Jochen Setz and Jorge Marques Pelizzoni. With PW, new principles can be automatically compiled from declarative specifications (formulas in first-order logic) without any knowledge in Mozart/Oz constraint programming.

Taken together, the XDK provides the following:

  • compilers for grammar files in various input languages (e.g. XML)
  • a solver for XDG parsing/generation
  • output functors to visualize the solutions of the solver
  • the principle compiler PrincipleWriter (PrincipleWriter) for automatic compilation of principles into Mozart/Oz constraint functors
  • lots of handcrafted example grammars
  • an adaptation of the large-scale English TAG grammar developed in the XTAG project
  • many useful shell-scripts

2.3 Literature

XDG is described most thoroughly in Ralph Debusmann's dissertation Extensible Dependency Grammar - A Modular Grammar Formalism Based On Multigraph Description. The XDK has been published in The XDG Grammar Development Kit. These and other papers can be found on the references pages of this document: References. Still more material can be found online here: http://www.ps.uni-sb.de/~rade/xdg.html. The latter page also includes pointers to slides, e.g. from the ESSLLI 2004 course by Ralph Debusmann and Denys Duchier: http://www.ps.uni-sb.de/~rade/talks.html.

2.4 Structure of this Manual

The structure of this manual is as follows. Installation explains how to install the XDK. Compiler explains how to write grammars. Grammars describes the example grammars included in the XDK distribution. This excludes the XDK version of the XTAG grammar, which is described in XTAG. In Solver, we explain the constraint-based solver for parsing and generation. Oracles is about oracles to guide the search for solutions. Outputs explains the various available output functors and how to define new ones. PrincipleWriter introduces the principle compiler. In Programs, we explain how to use the executable programs provided by the XDK. Debug illustrates how XDG grammars can be debugged. Changes lists the changes the XDK has gone through during its lifetime, and Contributors those people who have contributed to it. References contains pointers to all relevant papers, and Concept index is the concept index of this document.

Throughout this manual, sections labeled “developers only” contain information useful only for people who wish to understand the inner workings of the XDK. Here, in Directories, we describe the directory structure of the XDK. In Exceptions we specify the standard format for exceptions throughout the system. In Variable names, we describe the variable name conventions.
